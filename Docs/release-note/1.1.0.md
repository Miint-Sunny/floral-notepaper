## v1.1.0

### 终于不用自己翻 Release 了 | Highlight

好消息：你养的花会自己换新花盆了。

现在你可以直接在设置里检查、下载并安装更新，不用再手动打开 GitHub Release、找安装包、下载、覆盖安装。这一串流程终于可以少点仪式感了。

#### 应用内更新，爽

新增的更新功能支持从 GitHub Release 和 MirrorChyan 获取版本信息，并提供自动检查、手动检查、下载、安装和取消等完整流程。

你也可以在设置中调整自动检查频率，或者填写 MirrorChyan CDK 来使用对应的下载源。安装更新前，花笺还会尝试保存未保存的笔记，不必再为忘记 Ctrl+S 慌张。

Windows 更新链路也补上了安装回滚、注册表备份、进程校验和暂存目录保护等兜底。虽然不敢说从此万无一失，但至少摔下来的姿势会讲究一些。

这次还把应用配置与笔记数据拆到了各自的目录，并为旧版默认目录、自定义笔记目录和迁移失败后的回退补上了保护。升级时该搬的会搬，搬不动也不会急着拆旧家。

> ⚠️ 注意：应用内更新与数据迁移涉及的链路较多。如果遇到异常，请先保留现场并提交 [Issue](https://github.com/Achilng/floral-notepaper/issues)。

#### 桌面是你的，墙纸也该是

主窗口现在支持自定义背景图片，你可以在设置中选择、调整或移除背景，让花笺更像自己桌面的一部分。

通过全局快捷键打开便签时，窗口还可以跟随鼠标位置出现；快捷便签的“打开”页也加入了笔记搜索。少找几秒，灵感就少丢一点。

#### 写笔记，少一点打断

Markdown 预览新增了 HTML、GFM Alerts、代码块语言标签和文章内标题跳转支持。编辑器也补上了 Tab 缩进、右键菜单、图片粘贴与拖拽、撤销与重做，以及直接拖入 `.md`、`.markdown` 和 `.txt` 文件的能力。

错误提示改成了不挡路的 Toast，自动保存期间继续输入也不会再把新内容覆盖掉。很多功能单看都不大，但加起来就是一句：本该如此。

#### macOS 终于更像 macOS 了

macOS 主窗口改用原生红绿灯，并补齐了全屏关闭、绿灯退出全屏、Dock 图标恢复窗口和窗口重建等行为。Windows 和 Linux 仍保留原有的自定义窗口控件，各个平台各长各的，不再硬穿同一件外套。

发布流程也新增了 Linux 的 DEB、RPM 和 AppImage 构建产物。能不能成为主力另说，至少现在有门可以进了。

### No More Manual Release Hunting | Highlight

Good news: your flower can move itself into a newer pot now.

You can now check, download, and install updates directly from Settings instead of opening GitHub Releases, finding the right installer, downloading it, and running it yourself. That little ritual is finally optional.

#### In-app updates are here

The new update system fetches release information from GitHub Releases or MirrorChyan and supports automatic checks, manual checks, downloads, installation, and cancellation.

You can configure the check frequency or provide a MirrorChyan CDK for its download source. Before installation, Floral Notepaper will also try to save unsaved notes, so there is less reason for Ctrl+S anxiety.

The Windows update path now includes rollback, registry backup, process validation, and staging-directory safeguards. It is not a promise that nothing can ever fail, but the landing should be much less dramatic.

Application settings and note data now live in separate directories. Migration safeguards cover legacy defaults, custom note directories, and rollback when a move cannot be completed, so an upgrade will not discard the old home before the new one is ready.

> ⚠️ Note: In-app updates and data migration touch many parts of the application. If something goes wrong, preserve the affected data and submit an [Issue](https://github.com/Achilng/floral-notepaper/issues).

#### Your desk, your background

The main window now supports custom background images. You can choose, adjust, or remove the background in Settings and make Floral Notepaper feel more at home on your desktop.

Notes opened through a global shortcut can follow the cursor, and the quick notepad's Open page now includes note search. A few seconds saved here may be one idea kept there.

#### Fewer interruptions while writing

Markdown preview now supports embedded HTML, GFM Alerts, code-block language labels, and in-document heading navigation. The editor also gains Tab indentation, a context menu, image paste and drag-and-drop, undo and redo, plus direct opening of dropped `.md`, `.markdown`, and `.txt` files.

Errors now appear as unobtrusive toast notifications, and typing during an automatic save no longer risks replacing newer text. Most of these changes are small on their own; together, they make the editor feel the way it should have all along.

#### macOS now feels more like macOS

The macOS main window now uses native traffic lights, with fixes for closing from fullscreen, leaving fullscreen with the green button, restoring the window from the Dock, and rebuilding windows. Windows and Linux keep the existing custom window controls, so each platform can behave like itself.

The release workflow now also produces Linux packages in DEB, RPM, and AppImage formats. It may not be everyone's daily driver yet, but there is now a proper door to walk through.
